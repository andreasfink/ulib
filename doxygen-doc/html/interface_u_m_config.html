Detailed Description

UMConfig is an object to hold the data of config files.

config files are having the following syntax:

commented out lines are starting with a hash

a group starts with a group statement and ends with a empty line

include "some-other-config-file"

group = group1 param = value param2 = value2

group = group2 param = value param2 = value2

alternative syntax (windows.ini file style)

include "some-other-config-file"

[group1] param = value param2 = value2

[group2] param = value param2 = value2

Which groups are valid and read is defined while reading. The individal parameters then end up in a dictionary.

You define also if a group can be defined only once or multiple times.

Typical usage sequence:

UMConfig *config = [[UMConfig alloc]initWithFileName:"myconfig.conf"]; [config allowSingleGroup:@"main"]; [config allowSingleGroup:@"subsection"]; [config disallowSingleGroup:@"no-longer-supported-subsection"]; [config allowMultiGroup:@"multisection"]; [config read];

groups which are in the config file but not defined are simply skipped if they are in the disallowed definition it throws an exception instead.
