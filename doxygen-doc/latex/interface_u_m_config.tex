\doxysubsection{Detailed Description}
\mbox{\hyperlink{interface_u_m_config}{U\+M\+Config}} is an object to hold the data of config files. 

config files are having the following syntax\+:\hypertarget{interface_u_m_config_autotoc_md0}{}\doxysubsection{commented out lines are starting with a hash}\label{interface_u_m_config_autotoc_md0}
\hypertarget{interface_u_m_config_autotoc_md1}{}\doxysubsection{a group starts with a group statement and ends with a empty line}\label{interface_u_m_config_autotoc_md1}
include \char`\"{}some-\/other-\/config-\/file\char`\"{}

group = group1 param = value param2 = value2

group = group2 param = value param2 = value2

alternative syntax (windows.\+ini file style)

include \char`\"{}some-\/other-\/config-\/file\char`\"{}

\mbox{[}group1\mbox{]} param = value param2 = value2

\mbox{[}group2\mbox{]} param = value param2 = value2

Which groups are valid and read is defined while reading. The individal parameters then end up in a dictionary.

You define also if a group can be defined only once or multiple times.

Typical usage sequence\+:

\mbox{\hyperlink{interface_u_m_config}{U\+M\+Config}} $\ast$config = \mbox{[}\mbox{[}\mbox{\hyperlink{interface_u_m_config}{U\+M\+Config}} alloc\mbox{]}init\+With\+File\+Name\+:"myconfig.\+conf\char`\"{}\mbox{]};
  \mbox{[}config allow\+Single\+Group\+:@\char`\"{}main\char`\"{}\mbox{]};
  \mbox{[}config allow\+Single\+Group\+:@\char`\"{}subsection\char`\"{}\mbox{]};
  \mbox{[}config disallow\+Single\+Group\+:@\char`\"{}no-\/longer-\/supported-\/subsection\char`\"{}\mbox{]};
  \mbox{[}config allow\+Multi\+Group\+:@\char`\"{}multisection"\mbox{]}; \mbox{[}config read\mbox{]};

groups which are in the config file but not defined are simply skipped if they are in the disallowed definition it throws an exception instead.
