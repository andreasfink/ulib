22 /* the implementation is like this:
23 
24 1. There is a mainListener thread running doing nothing else than listening on the port.
25  If there is a new connection coming in, it is creating a connection object, add it to the connections array
26  and fires of a listenerThread to sit on the tcp stream;
27 
28 2. the connectionListener thread does sit on the socket and constantly reads. If there is a complete request
29  which came in, it will pass it on to requests queue and calls the delegate for http processing. After he's done
30  the response is being sent and the connection might be closed depending of its HTTP/1.1 or HTTP/1.0.
31 
32 */
