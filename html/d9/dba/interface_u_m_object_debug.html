Detailed Description

The debugging version of UMObject.

UMObjectDebug is a object handled like UMObject but allows to log stuff during retain/release thats why its compiled specifically without ARC.

Only useful for debuggging. You would simply inherit your object from UMObjectDebug instead of UMObject so you can track where your objects gets finally deallocated which you can't trap with ARC enabled by setting breakpoints at retainDebug and releaseDebug or by overloading these methods.
